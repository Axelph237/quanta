# Grover's Algorithm
One of the most well-known quantum algorithms is Grover's unstructured database algorithm. The basic premise is that we want to find the output state that directly correlates to our desired input, encoded into an *oracle* function. The way in which we do this is by querying the *oracle* function and *diffusing* the result until the state is the one we are searching for.

## Phone A Friend
#### The Basic Idea
"What is an unstructured database?" you ask. Well what a wise question! It can be a little hard to explain, so let's do like we usually do and take a look at a more approachable example first:

Say you wanted to call a new friend, but this was long before contacts and smart phones. What would you do? Your best guess is to look them up in the phone book. *You know their name, you just want their associated phone number*. We can assume each person has one phone number, and vice-versa. In other words, this is a *1-to-1 database* of names $\rightarrow$ phone numbers. The issue is that if that the entries in this particular phonebook are in no particular order, and there is an entry for every person on the planet. That means it could take you looking through a total of 8 billion entries to find your new friend's number!

Luckily for us, we can use Grover's algorithm to reduce the time this search would take beyond our current means! With one caveat: we must be able to *describe* our set of data in a quantum **oracle**.

## Asking The Oracle

So what is a quantum oracle? Well, in short, it is a sort of **black box**. The oracle is a specialized gate built for the problem we are trying to solve. The gate represents a set of quantum operations that model the system we are trying to explore. 

Phone books are a bad example: they are a dataset of random entries, the names can be any string of characters, and phone numbers are pretty random as well. Because of this, phone books aren't actually a great unstructured database. A better example might be the structure of an atom, its potential states, or even **the rules and board of a sudoku game**.

## Diffusion
#### Inversion-About-The-Mean
Now that our target state has been marked, we need a way to reliably extract that state. If we measured our system right now, every state would have an equal probability (since the phase here does not change the square of the amplitude). To solve this, Lov Grover defines what is called the diffusion operator $D$.


The goal of the diffuser is referred to as **inversion-about-the-mean**. In other words, the diffuser takes the average of all states and reflects them about that average.

In our above graph, we saw that our wave equation looked like the following:
$$
\Psi = \frac{1}{\sqrt{8}} (|000\rangle + |001\rangle + |010\rangle + |011\rangle \class{text-quanta-secondary}{- |100\rangle} + |101\rangle + |110\rangle + |111\rangle)
$$
*(Note, $\frac{1}{\sqrt{8}} \approx 0.3535$)*

Now the sum of all the amplitudes is: $7 \frac{1}{\sqrt{8}} - \frac{1}{\sqrt{8}} = \frac{6}{\sqrt{8}}$. Taking the average of these we get: $\frac{6}{\sqrt{8}} \div 8 = \frac{6}{8\sqrt{8}} = \frac{3}{4\sqrt{8}} \approx 0.265$.

If we reflect all of these states about that average (or, find the difference between two times the average $\overline{\alpha}$ and the state's amplitude $\alpha_i$: $2\overline{\alpha} - \alpha_i$), we get the following:

See how the amplitude of the marked state $|100\rangle$ has increased and the others have all decreased? This is *exactly* what we want! If we were to measure this state now, we would have an extremely highly likelyhood of measuring the solution to our problem.

## Could You Repeat Yourself?
#### $\frac{\pi}{4} \sqrt{N}$ times

There is just one problem, although our amplitude has increased, it still only has a total measurable probability of $\sim 0.88^2 \approx 0.77$. This is better than the probability of $0.125$ we would have had if we measured the initial state, but we can do better! Remember how at the beginning of the lesson we mentioned "repetitions" of $O$ and $D$? Well, it is those repetitions that will ultimately get us a much higher probability of measuring the correct state.

So how many times do we need to repeat this process? In his original paper, Lov Grover proved that we need $\frac{\pi}{4} \sqrt{N}$ iterations (we will skip the proof for now), where $N$ is the number of possible states. In our case, that's $2^3 = 8$ since we have three qubits in our system. So, $\frac{\pi}{4} \sqrt{8} \approx 2.22$. Since we can't have a fraction of an iteration, we round to the nearest integer, which is 2. 

After applying the oracle $O$ (marking the target state) and the defuser $D$ one more time (for a total of two times), we get the following amplitudes:

With a probability amplitude of $\sim 0.97$, we have a $0.97^2 \approx 0.94$ chance of measuring the correct state! To be sure we know the correct state, we might run the algorithm multiple times, keeping track of our outcome each time. That way we will measure the correct state the majority of the time!

## Scratch Paper
#### From Algorithm to Implementation

Okay, so we have looked at the steps to this algorithm, but how would we actually implement this on a quantum computer? For this section, let's scratch the paper, and instead look at the actual circuit.

**The Oracle** is an arbitrary unitary gate that is defined on a per-problem basis. Essentially, this is our "input" where we describe the database and constraints we want to solve for. Like with the sudoku example, this would be our board state and the rules of sudoku described by a matrix (check out THIS paper to see how this can be implemented for a game of sudoku).

**The Diffuser** is standard across all applications of Grover's algorithm. It is defined as:
$$
    D = 2|s\rangle\langle s| - I
$$
